## Project Overview

- **Type:** Monolith — a shell-orchestrated build & publish pipeline (infrastructure/CI project, not an application)
- **Primary Language:** Bash (compiles upstream components written in Go, Rust, and C)
- **Architecture:** Staged pipeline — bootstrap → preflight → toolchain → per-component build → package (nFPM) → reprepro APT repo → GitHub Pages publish
- **Repository:** `podman-ubuntu`

## Quick Reference

- **What it does:** Compiles the Podman container stack (12 packaged components) from upstream source on Ubuntu 24.04/26.04 (amd64 + arm64), packages them into `.deb`s with nFPM, and publishes a GPG-signed reprepro APT repository to GitHub Pages with three tracks (stable / edge / nightly).
- **Entry point:** `setup.sh` (full build); `scripts/build_<component>.sh` (single component); `scripts/package_all.sh` (staging → `.deb`); `scripts/ci_publish.sh` (repo assembly).
- **Config source of truth:** `config.sh` (env-driven, `${VAR:-default}` throughout).
- **Track selection:** purely by environment — stable = `source versions-stable.env`, edge = no env, nightly = `NIGHTLY_BUILD=true SHALLOW_CLONE=false`.
- **Build platform:** Linux only (dev host is macOS → use `bash -n` + Lima VMs `ubuntu-24` / `ubuntu-26`).
