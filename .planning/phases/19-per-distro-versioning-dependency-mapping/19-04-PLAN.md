<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| /etc/os-release + ldd/dpkg output (build host) → recorded baseline / declared deps | verify_depends.sh consumes detector output (host package DB) to assert the baseline |
| built .deb + 26.04 archive → installed system (container) | smoke_install_2604.sh apt-installs a locally-built .deb, resolving declared deps from the 26.04 archive inside a throwaway container |
| container runtime selection (SMOKE_RUNTIME/SMOKE_IMAGE env) | env-overridable runtime/image strings feed `docker`/`podman run` invocations |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-09 | Tampering | SMOKE_RUNTIME / SMOKE_IMAGE env interpolation into container run | mitigate | Validate SMOKE_RUNTIME is exactly `docker` or `podman` and SMOKE_IMAGE matches an image-name pattern before use; quote all expansions; the container is throwaway (`--rm`) so a compromised image cannot persist on the build host. |
| T-19-10 | Integrity | false-pass on 24.04 equivalence (t64 trap) | mitigate | Baseline encodes the documented t64 substitution explicitly (RESEARCH Pitfall 1); an undocumented dep delta on 24.04 FAILS rather than silently passing, so the equivalence check cannot rubber-stamp a wrong detected set. |
| T-19-11 | Repudiation | silent skip when no container runtime present | mitigate | smoke_install_2604.sh hard-errors (exit non-zero) when no runtime is available rather than skipping, so the PKG-08 gate cannot be silently bypassed. |
| T-19-SC | Tampering | apt-get install pulling deps from the 26.04 archive | accept | Standard apt over the distro's signed archive inside a `--rm` container; no new registry package is introduced by this project. The install is the test, not a production install. Low risk, throwaway environment. |
</threat_model>
