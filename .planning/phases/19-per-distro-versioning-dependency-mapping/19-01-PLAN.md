<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| build env → version string / filenames | `DISTRO` env var and `/etc/os-release` `VERSION_ID` are interpolated into the package version string and (transitively) `.deb` filenames |
| build host package DB → declared deps | `dpkg-query -S` output (owning package names) flows into package control metadata via the detector |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-01 | Tampering | detect_distro_version_id (DISTRO / VERSION_ID interpolation) | mitigate | Regex-validate the value against `^[0-9]+\.[0-9]+$` before composing VERSION_SUFFIX; fail closed on mismatch (RESEARCH Security V5). A malformed DISTRO cannot inject shell or path characters into the version string. |
| T-19-02 | Elevation (theoretical) | detect_runtime_depends running `ldd` | accept | `ldd` may invoke the dynamic loader; only ever run on freshly-built, in-tree binaries under DESTDIR (never external input). No untrusted binary reaches this function. Documented constraint; low risk. |
| T-19-03 | Repudiation / Integrity | detect_runtime_depends unmapped soname | mitigate | D-03 hard-fail: any resolved `.so` with no owning package aborts the build with a specific error, preventing a mislinked binary from shipping with a silently-wrong dependency set. |
| T-19-04 | Tampering | word-splitting / unquoted path expansion in detection loop | mitigate | `set -euo pipefail` is project-wide; all path expansions (`${lib}`, `${bin}`) are quoted; `realpath` normalizes before `dpkg-query`. |
</threat_model>
