Close verification GAP 1 (CR-01, BLOCKER): make `add_byhash_and_resign` in
`scripts/repo_byhash.sh` robust against the inherited `set -euo pipefail` so a
benign non-zero pipe head (e.g. a `sha256sum`/`awk` failure, a missing listed
index file, a transient `cp` error) can no longer abort the function after it has

half-signed repository to GitHub Pages. Fold in two cheap WARNING fixes that touch
the same signing/bootstrap surface (WR-01 anchored key extraction, WR-03 quoted
realpath bootstrap).

publish. A mid-function abort defeats that guarantee. This plan removes the abort
path and proves the helper survives a pipefail-triggering condition.

fingerprint extraction in `repo_manage.sh`; quoted realpath bootstrap in

    `GPG_KEY_ID=$(gpg --list-secret-keys --with-colons | awk -F: '/^fpr:/{print $10; exit}')`.
