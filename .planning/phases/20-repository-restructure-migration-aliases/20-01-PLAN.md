|----------|-------------|

| build env -> signed Release | `SignWith: yes` selects the single imported GPG key that signs every suite |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

Task 2 acceptance criterion `grep -q '^Suite: stable$'` enforces this. |

| T-20-03 | Elevation/Spoofing | Routing helper input | mitigate | `resolve_publish_targets`/`is_valid_suite` reject any track/distro/suite outside the explicit whitelist (Task 1), preventing a malformed publish target from creating an unintended suite. |
| T-20-SC | Tampering | npm/pip/cargo installs | accept | No package-manager installs in this plan (config + metadata + bash tests only); RESEARCH Package Legitimacy Audit: N/A. |
