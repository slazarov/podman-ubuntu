# Phase 22 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

> **Scope note (read first).** This phase has only ONE visual surface: the generated
> `index.html` landing page (emitted by `scripts/ci_publish.sh` Step 5, heredoc at
> ~L472–611). It is a single static page with an **already-shipped inline design
> system** — no framework, no shadcn, no React, no component library, and none should
> be introduced. The phase's UI work is a *targeted extension* of that existing page:
> add a distro toggle (D-07/D-08), rewrite the track snippets to DEB822 (RESEARCH OQ1),
> and add a deprecation callout (D-09). `docs/apt-repository.md` is GitHub-rendered
> Markdown — it has no custom CSS and is governed only by the Copywriting Contract below.
>
> **Governing principle:** this contract documents the values the page **actually ships
> today**, not idealized defaults. New elements conform to the clean spacing scale;
> pre-existing values that violate it are recorded as documented exceptions (out of
> this phase's scope to restyle). The executor extends the page — it does not restyle it.

## Spacing Scale

**New elements** (`.distro-btn`, the distro toggle row, the deprecation callout) MUST
conform to this clean scale. CSS is rem-based on the page; the px equivalents below
assume the browser-default 16px root.

| Token | rem | px | Usage on this page |
|-------|-----|-----|--------------------|
| sm | 0.5rem | 8px | Button vertical padding (`.tab-btn` / `.distro-btn`), tight gaps |
| md | 1rem | 16px | Default block spacing, `pre` padding, tab gap |
| lg | 1.5rem | 24px | Button horizontal padding (`.tab-btn` / `.distro-btn`), section margins |

**Contract for new elements:** `.distro-btn` reuses `.tab-btn`'s box metrics exactly —
`padding: 0.5rem 1.5rem` (8px / 24px, both on-scale). The distro toggle row sits above
the track tabs with `margin: 1.5rem 0` (24px), matching `.tab-group`. The deprecation
callout uses `padding: 1rem` (16px) and `margin: 1.5rem 0` (24px).

**Documented exceptions (pre-existing, OUT OF SCOPE to change this phase):**
- Table cells: `padding: 0.4rem 0.8rem` (6.4px / 12.8px) — off-scale, pre-existing, leave as-is.
- `code` inline padding: `0.15em 0.4em` — em-based, pre-existing, leave as-is.
- Body margin: `2rem auto` (32px) — on-scale, pre-existing, keep.

Rationale for exceptions: this phase **extends** the page, it does not restyle it.
Re-tokenizing the existing table/code CSS is out of scope and would be visual churn.

## Typography

The page uses **em-relative** font sizing against a 1rem (16px) root with the system
font stack. These are the **actual shipped values** — the executor matches them so new
elements read as part of the same page.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 1rem (16px) | 400 (normal) | 1.6 |
| Heading (h1) | browser default (~2em) | bold (UA default) | 1.6 (inherited) |
| Button / tab / control | 0.95em (~15px) | 400; **600 when `.active`** | 1.6 (inherited) |
| Small / meta (table, code, links-in-headings) | 0.8–0.9em | 400 | 1.6 (inherited) |

**Contract for new elements:**
- `.distro-btn` text: `font-size: 0.95em`, weight 400, weight **600 when active** —
  identical to `.tab-btn`. Do NOT introduce a new size or weight.
- Deprecation callout body: inherits body (1rem / 400 / 1.6).

**Documented exception:** body line-height is **1.6**, not the template's suggested 1.5.
This is the shipped value; new content inherits it. Do not change it to 1.5 — that would
make new blocks line up differently from the rest of the page.

Weight count: **2 weights total** (400 normal, 600 for active controls / UA-bold
headings). Size set is em-relative rather than a fixed 3–4px ladder — accepted as the
existing page idiom; no new sizes added by this phase.

## Color

The page ships a near-monochrome neutral palette with a single green accent. These are
the **actual shipped hex values**. New elements use only colors already in this table.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` (page bg) + text `#1a1a1a` | Page background and body text |
| Secondary (30%) | `#f4f4f4` | Code/pre backgrounds, table headers, **inactive** tab/distro buttons |
| Accent (10%) | `#2ea44f` (green) | Reserved — see list below |
| Link | `#0366d6` (blue) | Hyperlinks only (pre-existing; not a general accent) |
| Border | `#ddd` / `#333` | Card/table/button borders (`#ddd`); h1 underline + active-control edges (`#333`) |
| Destructive | not applicable | No destructive UI on this static page |

**Accent (`#2ea44f`) reserved for — exhaustive list, do NOT extend:**
- The `recommended` track card border (`.track.recommended`).
- The `(recommended)` label text appended to the stable track heading.

**Hard constraint for the distro toggle (D-07/D-08):** the toggle's **active** state
reuses the existing active-tab treatment — white background (`#fff`) + font-weight 600 +
`#333`/`#ddd` border edges — exactly as `.tab-btn.active`. The toggle MUST NOT introduce
the green accent or any new color. Green stays reserved for the "recommended" marker so
its 10% meaning is preserved. Inactive distro buttons use `#f4f4f4` (secondary), matching
inactive tabs.

## Interaction Contract (distro toggle — MIGR-02 / D-07/D-08)

| Aspect | Contract |
|--------|----------|
| Placement | Two buttons (`.distro-btn`) in a row **above** the existing track tabs, under the "Setup" / "Add the repository" step. |
| Default state | **Ubuntu 24.04** active on load (majority of existing users). The 2404 snippets render visible; 2604 snippets hidden. |
| Mechanism | Vanilla JS `setDistro(ver)` extending the existing `showTab()` pattern (L601–608). Toggles `.active` on the clicked `.distro-btn` and shows/hides snippet blocks by a `data-distro="2404"`/`"2604"` attribute. |
| Composition | Each of the three track tabs (`stable`/`edge`/`nightly`) contains **two** snippet blocks — one per distro — differing only in the `Suites:` line (`<track>-2404` vs `<track>-2604`). `setDistro` swaps which is visible across all tabs at once. |
| State persistence | **Ephemeral** (no `localStorage`). Discretion item D-07; locked here as ephemeral — resets to 24.04 on reload. |
| Active styling | Reuses `.tab-btn.active` treatment exactly (`#fff` bg, weight 600, `#333`/`#ddd` edges). No new accent. |
| Accessibility | Buttons are real `<button>` elements (keyboard-focusable, Enter/Space activate by default). Keep that — do not swap to `<div onclick>`. |

**Snippet format (RESEARCH OQ1 — flagged dependency, planner/user to confirm; visual
treatment identical either way):** the track snippets should be **rewritten from the
current legacy one-line `deb [...]` form to DEB822** to match `docs/apt-repository.md`,
standardizing the keyring path on `/etc/apt/keyrings/podman-ubuntu.gpg`. This is a
content/path decision, not a visual one — both forms render in the same `<pre><code>`
block with identical styling. The UI-SPEC does not block on OQ1; it records that the
chosen format must be **consistent between index.html and the docs**.
