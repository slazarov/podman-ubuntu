**mold activation in 2 Rust build scripts:**

For each of: build_netavark.sh, build_aardvark_dns.sh -- add mold configuration AFTER the existing sccache block (after the `fi` that closes the sccache check), still inside the "Configuring Cargo optimization" step:

```bash
# Enable mold linker for faster Rust linking when configured
if [[ "${MOLD_ENABLED:-false}" == "true" ]] && command -v mold &>/dev/null; then
    # Use project-level cargo config to avoid conflicts with sccache RUSTC_WRAPPER
    mkdir -p .cargo
    cat > .cargo/config.toml << 'TOML'
[target.'cfg(target_os = "linux")']
rustflags = ["-C", "link-arg=-fuse-ld=mold"]
TOML
    echo "  mold linker enabled for Rust compilation"
fi
```

IMPORTANT: Using .cargo/config.toml (not RUSTFLAGS env var) is deliberate. The env var conflicts with sccache's RUSTC_WRAPPER. The config.toml approach works correctly with both sccache and mold enabled simultaneously. The `cfg(target_os = "linux")` target selector is architecture-agnostic -- works on both amd64 and arm64.
