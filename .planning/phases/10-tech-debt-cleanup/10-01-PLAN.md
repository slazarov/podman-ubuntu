Add conditional removal of mold and clang apt packages to uninstall.sh. This ordering matters because apt-get remove should run before /etc/containers is deleted (see Research pitfall 5).

Add the following block between the ccache cache cleanup and the configuration directory removal:

```bash
# Remove mold and clang apt packages (installed by MOLD_ENABLED=true)
if dpkg -s mold &>/dev/null; then
    apt-get remove -y mold
    REMOVED+=("apt package: mold")
else
    SKIPPED+=("apt package: mold (not installed)")
fi

if dpkg -s clang &>/dev/null; then
    apt-get remove -y clang
    REMOVED+=("apt package: clang")
else
    SKIPPED+=("apt package: clang (not installed)")
fi
```

This follows the existing REMOVED/SKIPPED tracking pattern used throughout uninstall.sh. Use `dpkg -s` to check if the package is installed before attempting removal (avoids apt errors on systems where MOLD_ENABLED was never true). Use `apt-get remove -y` (not purge) to match the project's removal semantics. DEBIAN_FRONTEND=noninteractive is already exported at line 4 of uninstall.sh, so no additional export is needed.

Do NOT add ccache apt package removal -- it was not flagged by the audit and is not in the phase requirements.
