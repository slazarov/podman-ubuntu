<task type="auto">
  <name>Task 2: Create repository management script</name>
  <files>scripts/repo_manage.sh</files>
  <action>
Create scripts/repo_manage.sh following project shell script conventions (set -euo pipefail, toolpath resolution, source config.sh and functions.sh, error_handler trap).

**Usage:** `repo_manage.sh <suite> <deb-directory> [output-directory]`
- suite: "stable" or "edge"
- deb-directory: path containing .deb files (typically output/)
- output-directory: where to create the repository structure (default: repo-output/)

**Script logic:**

1. **Parse and validate arguments:**
   - Require suite as first arg, validate it is "stable" or "edge"
   - Require deb-directory as second arg, validate it exists and contains .deb files
   - Optional third arg for output directory (default: ${toolpath}/repo-output)

2. **GPG key import (CI support):**
   - If GPG_PRIVATE_KEY env var is set, import it: `echo "${GPG_PRIVATE_KEY}" | gpg --batch --import`
   - After import, set ownertrust to ultimate: extract fingerprint with `gpg --list-keys --with-colons | grep fpr | head -1 | cut -d: -f10`, then `echo "${GPG_KEY_ID}:6:" | gpg --batch --import-ownertrust`
   - If GPG_PRIVATE_KEY is NOT set, verify at least one secret key exists in the keyring: `gpg --list-secret-keys --with-colons 2>/dev/null | grep -q "^sec:"` or exit with error

3. **Prepare reprepro base directory:**
   - Create ${OUTPUT_DIR}/conf/
   - Copy packaging/repo/conf/distributions and packaging/repo/conf/options to ${OUTPUT_DIR}/conf/

4. **Add packages:**
   - Loop over all .deb files in the deb-directory
   - For each: `reprepro -Vb "${OUTPUT_DIR}" includedeb "${SUITE}" "${deb_file}"`
   - Track count of added packages

5. **Export metadata (generates InRelease + Release.gpg):**
   - `reprepro -b "${OUTPUT_DIR}" export`

6. **Copy public GPG key to repository root:**
   - If packaging/repo/pubkey.gpg exists, copy it to ${OUTPUT_DIR}/podman-debian.gpg
   - If not, export from keyring: `gpg --export $(gpg --list-keys --with-colons | grep fpr | head -1 | cut -d: -f10) > "${OUTPUT_DIR}/podman-debian.gpg"`

7. **Cleanup reprepro internals (not needed for serving):**
   - Remove ${OUTPUT_DIR}/db/ directory
   - Remove ${OUTPUT_DIR}/conf/ directory

8. **Summary output:**
   - Print suite name, package count, output directory
   - List contents of dists/${SUITE}/ to confirm InRelease and Release.gpg exist

Make the script executable (chmod +x).

<task type="auto">
  <name>Task 3: Create user setup documentation</name>
  <files>docs/apt-repository.md</files>
  <action>
Create docs/apt-repository.md with user-facing instructions for adding the APT repository.

**Structure:**

1. **Title and overview:** Brief explanation that this is a custom APT repository for Podman compiled from source for Ubuntu 24.04.

2. **Quick Start (under 5 commands):**
   ```bash
   # Download the GPG signing key
   sudo mkdir -p /etc/apt/keyrings
   sudo wget -qO /etc/apt/keyrings/podman-debian.gpg \
     https://slazarov.github.io/podman-debian/podman-debian.gpg

   # Add the repository (DEB822 format)
   sudo tee /etc/apt/sources.list.d/podman-debian.sources << 'EOF'
   Types: deb
   URIs: https://slazarov.github.io/podman-debian
   Suites: stable
   Components: main
   Signed-By: /etc/apt/keyrings/podman-debian.gpg
   EOF

   # Update and install
   sudo apt update
   sudo apt install -y podman-suite
   ```

3. **Using the edge suite:** Same instructions but with `Suites: edge` in the .sources file. Explain that edge tracks latest upstream tags.

4. **Installing individual packages:** List available packages (podman-podman, podman-crun, etc.) and show `apt install podman-podman` example.

5. **Supported architectures:** amd64 and arm64.

6. **Troubleshooting:** Common issues (GPG key not found, signature errors) with solutions.

Do NOT use apt-key (deprecated). Do NOT use one-line sources.list format (legacy). Use only DEB822 .sources format with signed-by.
