Create `scripts/package_all.sh` -- the orchestrator that iterates all components, extracts versions, and invokes nFPM to produce .deb packages.

The script must:

1. Follow established conventions: `set -euo pipefail`, error trap, toolpath detection, source config.sh and functions.sh (same boilerplate as other scripts in scripts/).

2. Define an `extract_version()` function that handles all 3 version patterns:
```bash
extract_version() {
    local tag="$1"
    local component="$2"

    case "$component" in
        pasta)
            # Date-based: already numeric (e.g., 20250302)
            echo "${tag}"
            ;;
        container-configs)
            # Namespaced tag: common/v0.67.0 -> 0.67.0
            echo "${tag}" | sed 's|^.*/v||'
            ;;
        *)
            # Standard: strip v prefix (v5.5.2 -> 5.5.2)
            echo "${tag#v}"
            ;;
    esac
}
```

3. Define component-to-tag mapping using an associative array that maps component names to their config.sh tag variables. The tag variables are already exported by config.sh when it's sourced. The mapping:
```bash
declare -A COMPONENT_TAGS=(
    ["podman"]="${PODMAN_TAG}"
    ["crun"]="${CRUN_TAG}"
    ["conmon"]="${CONMON_TAG}"
    ["netavark"]="${NETAVARK_TAG}"
    ["aardvark-dns"]="${AARDVARK_DNS_TAG}"
    ["pasta"]=""  # Special: uses date-based version, handled separately
    ["fuse-overlayfs"]="${FUSE_OVERLAYFS_TAG}"
    ["catatonit"]="${CATATONIT_TAG}"
    ["buildah"]="${BUILDAH_TAG}"
    ["skopeo"]="${SKOPEO_TAG}"
    ["toolbox"]="${TOOLBOX_TAG}"
    ["container-configs"]="${CONTAINER_LIBS_TAG}"
)
```
For pasta, the version should be the current date (matching build_pasta.sh's `export GIT_CHECKED_OUT_TAG=$(date +"%Y%m%d")`). The orchestrator should use the same date format: `$(date +"%Y%m%d")`.

4. Validate prerequisites:
   - Check that DESTDIR is set and the staging tree exists
   - Check that nfpm is available in PATH (if not, suggest `go install github.com/goreleaser/nfpm/v2/cmd/nfpm@v2.45.0`)
   - Check that packaging/nfpm/ directory exists with YAML files

5. Create output/ directory (`mkdir -p "$OUTPUT_DIR"`).

6. Iterate all 12 components in order:
```bash
COMPONENTS=(
    "podman"
    "crun"
    "conmon"
    "netavark"
    "aardvark-dns"
    "pasta"
    "fuse-overlayfs"
    "catatonit"
    "buildah"
    "skopeo"
    "toolbox"
    "container-configs"
)
```

For each component:
- Get the tag from COMPONENT_TAGS (special case: pasta uses date)
- Call extract_version() to clean the tag
- Append `~podman1` suffix
- Export VERSION, ARCH, and DESTDIR environment variables
- Call `nfpm pkg --config "${NFPM_DIR}/${component}.yaml" --target "${OUTPUT_DIR}" --packager deb`
- Print progress: `">>> Packaging: podman-${component} (${VERSION})"`
- Print completion: `">>> Done: podman-${component}"`

7. After all components, package the meta-package:
- Use podman's version for the suite package (per discretion recommendation)
- Export VERSION (podman version with ~podman1 suffix)
- Call `nfpm pkg --config "${NFPM_DIR}/suite.yaml" --target "${OUTPUT_DIR}" --packager deb`

8. Print summary at the end: count of .deb files in output/, list their names and sizes.

9. Error handling: `set -euo pipefail` with error_handler trap means if any nFPM invocation fails, the script stops immediately. Add a clear message identifying which component failed. Use the existing error_handler pattern.

**Script structure outline:**
```
#!/bin/bash
set -euo pipefail
[toolpath detection boilerplate]
source config.sh
source functions.sh
trap 'error_handler $? $LINENO "$BASH_SOURCE"' ERR

NFPM_DIR="${toolpath}/packaging/nfpm"
OUTPUT_DIR="${toolpath}/output"

[extract_version function]
[validate prerequisites]
[component-tag mapping]
[iterate and package]
[package meta-package]
[summary output]
```

Make the script executable (chmod +x).

**Important:** The script should NOT build or stage anything -- it assumes build scripts have already run with DESTDIR set and the staging tree is populated. The script only does the nFPM packaging step.

**DESTDIR handling:** The script expects DESTDIR to be set in the environment when invoked. Example usage:
```bash
export DESTDIR=/tmp/podman-staging
# ... run build scripts with DESTDIR set ...
./scripts/package_all.sh
```
