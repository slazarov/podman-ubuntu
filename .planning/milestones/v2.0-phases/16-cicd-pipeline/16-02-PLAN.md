Create `.github/workflows/build-packages.yml`. Create the `.github/workflows/` directory first.

The workflow has three jobs: build-amd64, build-arm64, and publish. Use the research document's architecture but with the project's own `ci_publish.sh` for the publish step.

**Workflow-level configuration:**

```yaml
name: Build and Publish Packages

on:
  workflow_dispatch:
    inputs:
      build_track:
        description: 'Build track (stable = pinned versions, edge = latest upstream)'
        required: true
        type: choice
        options:
          - stable
          - edge
        default: stable

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: "pages"
  cancel-in-progress: false
```

**build-amd64 job** (runs-on: ubuntu-24.04, timeout-minutes: 180):

Steps:
1. `actions/checkout@v4`

2. **Set version tags** (conditional on build track):
   ```yaml
   - name: Set version tags (stable)
     if: inputs.build_track == 'stable'
     run: |
       # Source pinned versions into GITHUB_ENV for all subsequent steps
       while IFS= read -r line; do
         # Skip comments and empty lines
         [[ "$line" =~ ^#.*$ || -z "$line" ]] && continue
         # Strip 'export ' prefix and add to GITHUB_ENV
         echo "${line#export }" >> "$GITHUB_ENV"
       done < versions-stable.env
   ```
   For edge: no version step needed. Empty TAG variables in config.sh cause `git_checkout ""` to fetch the latest tag automatically via `get_latest_tag`.

3. **Build all components:**
   ```yaml
   - name: Build all components
     run: |
       export DEBIAN_FRONTEND=noninteractive
       export DESTDIR="${{ runner.temp }}/podman-staging"
       mkdir -p "$DESTDIR"
       sudo -E ./setup.sh
     timeout-minutes: 150
   ```
   Note: `sudo -E` preserves environment (DESTDIR, *_TAG vars). The TAG variables from GITHUB_ENV are automatically in the environment.

4. **Install nFPM:**
   ```yaml
   - name: Install nFPM
     run: |
       # Use Go installed by setup.sh
       export PATH="/opt/go/$(ls /opt/go/)/bin:$HOME/go/bin:$PATH"
       go install github.com/goreleaser/nfpm/v2/cmd/nfpm@v2.45.0
   ```

5. **Package with nFPM:**
   ```yaml
   - name: Package all components
     run: |
       export PATH="/opt/go/$(ls /opt/go/)/bin:$HOME/go/bin:$PATH"
       export DESTDIR="${{ runner.temp }}/podman-staging"
       ./scripts/package_all.sh
   ```

6. **Upload artifacts:**
   ```yaml
   - uses: actions/upload-artifact@v4
     with:
       name: debs-amd64
       path: output/*.deb
       retention-days: 30
   ```

**build-arm64 job** (runs-on: ubuntu-24.04-arm, timeout-minutes: 180):

Identical steps to build-amd64 except:
- `runs-on: ubuntu-24.04-arm`
- Artifact name: `debs-arm64`

To avoid YAML duplication, do NOT use a matrix (research warns against fail-fast issues with matrix). Instead, duplicate the job definition with the two differences noted above. This is explicit and avoids matrix complexity.

**publish job** (runs-on: ubuntu-24.04):

```yaml
publish:
  needs: [build-amd64, build-arm64]
  runs-on: ubuntu-24.04
  environment:
    name: github-pages
    url: ${{ steps.deployment.outputs.page_url }}
  steps:
    - uses: actions/checkout@v4

    - uses: actions/download-artifact@v4
      with:
        pattern: debs-*
        path: all-debs/
        merge-multiple: true

    - name: Install reprepro
      run: |
        sudo apt-get update
        sudo apt-get install -y reprepro

    - name: Build and publish repository
      env:
        GPG_PRIVATE_KEY: ${{ secrets.GPG_PRIVATE_KEY }}
      run: |
        ./scripts/ci_publish.sh \
          "${{ inputs.build_track }}" \
          "all-debs" \
          "https://${{ github.repository_owner }}.github.io/${{ github.event.repository.name }}" \
          "repo-output"

    - uses: actions/configure-pages@v4

    - uses: actions/upload-pages-artifact@v3
      with:
        path: repo-output/

    - id: deployment
      uses: actions/deploy-pages@v4
```

IMPORTANT details:
- Use `${{ github.repository_owner }}.github.io/${{ github.event.repository.name }}` instead of hardcoding the URL. This makes the workflow portable.
- `merge-multiple: true` in download-artifact combines both arch artifacts into one directory.
- The `environment: name: github-pages` block is required for the deploy-pages action.
- `needs: [build-amd64, build-arm64]` ensures atomic publishing (publish only runs if BOTH succeed).
