**1. `versions-stable.env`** at repository root.
A shell-sourceable file that exports pinned version tags for the stable build track. Use the SAME variable names as config.sh so they override the empty defaults when sourced before setup.sh. Note: crun and toolbox tags do NOT use v prefix (matches upstream convention). Pasta is not included because it uses date-based versioning computed at build time.

**2. `scripts/ci_publish.sh`** - CI-specific two-suite repository publisher.

Usage: `ci_publish.sh <suite> <deb-directory> <repo-url> <output-directory>`

Arguments:
- `suite`: "stable" or "edge" -- the suite being published with NEW packages
- `deb-directory`: path to directory containing freshly built .deb files
- `repo-url`: live repository URL (e.g., "https://slazarov.github.io/podman-debian")
- `output-directory`: where to create the final repository output

Script logic (follow the same coding patterns as repo_manage.sh -- set -euo pipefail, toolpath detection, source config.sh and functions.sh, error trap, argument validation with usage function):

Step 1: Determine the OTHER suite (`stable` -> `edge`, `edge` -> `stable`).

Step 2: Download existing .deb files from the OTHER suite on the live repo.
- Fetch `${repo-url}/dists/${OTHER_SUITE}/main/binary-amd64/Packages` and `binary-arm64/Packages` via curl.
- Parse each `Packages` file for `Filename:` lines to get relative paths to .deb files.
- Download each .deb file to a temporary directory (`other-suite-debs/`).
- Use `curl -sfL` with `|| true` so that failure (e.g., first-ever deploy with no live repo) is non-fatal.
- Print a count of how many packages were imported for the other suite.

Step 3: Run the existing `repo_manage.sh` for the CURRENT suite with the new .deb files.
- Invoke: `"${toolpath}/scripts/repo_manage.sh" "${SUITE}" "${DEB_DIR}" "${OUTPUT_DIR}"`
- This handles GPG import, reprepro, metadata export, pubkey copy, AND cleanup.

Step 4: If other-suite .deb files were downloaded (directory is non-empty), rebuild the conf/ and add them:
- `mkdir -p "${OUTPUT_DIR}/conf"` and copy `packaging/repo/conf/distributions` and `packaging/repo/conf/options` into it.
- For each .deb in `other-suite-debs/`: `reprepro -Vb "${OUTPUT_DIR}" includedeb "${OTHER_SUITE}" "$deb"`
- Run `reprepro -b "${OUTPUT_DIR}" export` to regenerate metadata for both suites.
- Clean up: `rm -rf "${OUTPUT_DIR}/db" "${OUTPUT_DIR}/conf"`

Step 5: Print summary showing both suites and package counts.

Make the script executable (chmod +x).

IMPORTANT: The script must handle the first-ever publish gracefully -- when there is no live repo, the curl downloads in Step 2 will fail, and that is OK (|| true). Step 4 will be skipped because the other-suite-debs directory will be empty. Only the current suite gets published.
