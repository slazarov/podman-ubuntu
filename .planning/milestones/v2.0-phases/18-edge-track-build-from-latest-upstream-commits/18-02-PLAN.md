Modify `scripts/ci_publish.sh` to handle three suites (stable, edge, nightly) instead of two. The key change: instead of a single "other suite", there are now TWO other suites to preserve.

1. **Update suite validation** (around line 69): Change the validation check from:
   ```bash
   if [[ "${SUITE}" != "stable" && "${SUITE}" != "edge" ]]; then
       echo "ERROR: Invalid suite ..."
   fi
   ```
   To:
   ```bash
   if [[ "${SUITE}" != "stable" && "${SUITE}" != "edge" && "${SUITE}" != "nightly" ]]; then
       echo "ERROR: Invalid suite '${SUITE}'. Must be 'stable', 'edge', or 'nightly'." >&2
       exit 1
   fi
   ```

2. **Update "other suites" logic** (around line 91): Replace the single OTHER_SUITE variable with an array of OTHER_SUITES. Instead of:
   ```bash
   if [[ "${SUITE}" == "stable" ]]; then
       OTHER_SUITE="edge"
   else
       OTHER_SUITE="stable"
   fi
   ```
   Use:
   ```bash
   ALL_SUITES=(stable edge nightly)
   OTHER_SUITES=()
   for s in "${ALL_SUITES[@]}"; do
       if [[ "$s" != "${SUITE}" ]]; then
           OTHER_SUITES+=("$s")
       fi
   done
   ```

3. **Update the download loop** (Step 2, around line 107): Wrap the existing download logic in an outer loop over `OTHER_SUITES`. For each other suite, download its packages to a temporary directory. Track per-suite counts for the summary.

   Replace the single-suite download with:
   ```bash
   declare -A OTHER_SUITE_DEBS_DIRS
   declare -A OTHER_SUITE_COUNTS
   total_other_count=0

   for other_suite in "${OTHER_SUITES[@]}"; do
       echo ">>> Downloading existing packages for '${other_suite}' suite..."
       other_dir=$(mktemp -d)
       OTHER_SUITE_DEBS_DIRS["${other_suite}"]="${other_dir}"
       suite_count=0

       for arch in amd64 arm64; do
           packages_url="${REPO_URL}/dists/${other_suite}/main/binary-${arch}/Packages"
           echo "  Fetching: ${packages_url}"
           packages_content=$(curl -sfL "${packages_url}" 2>/dev/null || true)

           if [[ -z "${packages_content}" ]]; then
               echo "  No Packages file for ${other_suite}/binary-${arch} (first deploy or not published)"
               continue
           fi

           while IFS= read -r filename; do
               if [[ -n "${filename}" ]]; then
                   deb_url="${REPO_URL}/${filename}"
                   deb_basename=$(basename "${filename}")
                   if [[ -f "${other_dir}/${deb_basename}" ]]; then
                       continue
                   fi
                   echo "  Downloading: ${deb_basename}"
                   if curl -sfL -o "${other_dir}/${deb_basename}" "${deb_url}"; then
                       suite_count=$((suite_count + 1))
                   else
                       echo "  WARNING: Failed to download ${deb_basename}, skipping" >&2
                       rm -f "${other_dir}/${deb_basename}"
                   fi
               fi
           done <<< "$(echo "${packages_content}" | grep "^Filename:" | sed 's/^Filename: *//')"
       done

       OTHER_SUITE_COUNTS["${other_suite}"]=${suite_count}
       total_other_count=$((total_other_count + suite_count))
       echo ">>> Downloaded ${suite_count} packages for '${other_suite}' suite"
       echo ""
   done
   ```

4. **Update Step 4** (add other suites' packages): Loop over `OTHER_SUITES` instead of a single suite. For each OTHER_SUITE with downloaded packages, run `reprepro includedeb`. Only rebuild conf/ and export once after all suites are added.

   ```bash
   if [[ ${total_other_count} -gt 0 ]]; then
       echo ">>> Adding other suites' packages to repository..."
       echo ""

       mkdir -p "${OUTPUT_DIR}/conf"
       cp "${REPO_CONF}/conf/distributions" "${OUTPUT_DIR}/conf/"
       cp "${REPO_CONF}/conf/options" "${OUTPUT_DIR}/conf/"

       for other_suite in "${OTHER_SUITES[@]}"; do
           suite_count=${OTHER_SUITE_COUNTS["${other_suite}"]}
           if [[ ${suite_count} -eq 0 ]]; then
               echo ">>> No packages for '${other_suite}' suite (first deploy or not published)"
               continue
           fi

           echo ">>> Adding '${other_suite}' suite packages..."
           other_added=0
           for deb_file in "${OTHER_SUITE_DEBS_DIRS["${other_suite}"]}"/*.deb; do
               if [[ -f "${deb_file}" ]]; then
                   echo "  Adding: $(basename "${deb_file}")"
                   reprepro -Vb "${OUTPUT_DIR}" includedeb "${other_suite}" "${deb_file}"
                   other_added=$((other_added + 1))
               fi
           done
           echo ">>> Added ${other_added} packages to '${other_suite}' suite"
           echo ""
       done

       echo ">>> Re-exporting repository metadata for all suites..."
       reprepro -b "${OUTPUT_DIR}" export
       echo ">>> Metadata exported"
       echo ""

       rm -rf "${OUTPUT_DIR}/db" "${OUTPUT_DIR}/conf"
       echo ">>> Cleaned up reprepro internals"
       echo ""
   else
       echo ">>> No packages for other suites (first deploy or no live repo)"
       echo ">>> Only '${SUITE}' suite will be published"
       echo ""
   fi

   # Clean up all temp dirs
   for other_suite in "${OTHER_SUITES[@]}"; do
       rm -rf "${OTHER_SUITE_DEBS_DIRS["${other_suite}"]}"
   done
   ```

<task type="auto">
  <name>Task 2: Add nightly option and cron trigger to GitHub Actions workflow</name>
  <files>.github/workflows/build-packages.yml</files>
  <action>
Modify `.github/workflows/build-packages.yml` to support nightly builds:

1. **Add cron schedule trigger**: Add a `schedule` trigger alongside the existing `workflow_dispatch`:
   ```yaml
   on:
     schedule:
       - cron: '30 4 * * *'  # 4:30 AM UTC daily — nightly builds
     workflow_dispatch:
       inputs:
         build_track:
           description: 'Build track (stable = pinned versions, edge = latest upstream tags, nightly = latest upstream commits)'
           required: true
           type: choice
           options:
             - stable
             - edge
             - nightly
           default: stable
   ```

2. **Add build_track resolution**: At the top of each build job (build-amd64, build-arm64), add a step that resolves the effective build track. Scheduled runs (cron) don't have `inputs.build_track`, so default to `nightly`:
   ```yaml
   - name: Resolve build track
     id: track
     run: |
       TRACK="${{ inputs.build_track || 'nightly' }}"
       echo "track=${TRACK}" >> "$GITHUB_OUTPUT"
       echo "Build track: ${TRACK}"
   ```
   Then replace ALL occurrences of `${{ inputs.build_track }}` with `${{ steps.track.outputs.track }}` in that job.

3. **Add nightly-specific environment** in the build step of each build job. After the existing stable version parsing block, add a nightly block. The nightly env values (NIGHTLY_BUILD=true, SHALLOW_CLONE=false) are inlined here rather than sourced from versions-nightly.env, to avoid sudo env sourcing complexity. versions-nightly.env serves as canonical documentation of these values.
   ```yaml
   - name: Build all components
     run: |
       DESTDIR="${{ runner.temp }}/podman-staging"
       mkdir -p "$DESTDIR"
       mkdir -p "$HOME/.cache/go-build" "$HOME/.cache/go-mod"
       ENV_ARGS="DEBIAN_FRONTEND=noninteractive DESTDIR=$DESTDIR"
       ENV_ARGS="$ENV_ARGS GOCACHE=$HOME/.cache/go-build GOMODCACHE=$HOME/.cache/go-mod"
       if [[ "${{ steps.track.outputs.track }}" == "stable" ]]; then
         while IFS= read -r line; do
           [[ "$line" =~ ^[[:space:]]*# || -z "$line" ]] && continue
           line="${line#export }"
           key="${line%%=*}"
           val="${line#*=}"
           val="${val%\"}"
           val="${val#\"}"
           ENV_ARGS="$ENV_ARGS ${key}=${val}"
         done < versions-stable.env
       elif [[ "${{ steps.track.outputs.track }}" == "nightly" ]]; then
         ENV_ARGS="$ENV_ARGS NIGHTLY_BUILD=true SHALLOW_CLONE=false"
       fi
       sudo env $ENV_ARGS ./setup.sh
       sudo chown -R "$(id -u):$(id -g)" "$HOME/.cache/go-build" "$HOME/.cache/go-mod"
     timeout-minutes: 150
   ```

   Note: For edge builds, no extra env vars are set (existing behavior: empty TAGs -> auto-detect latest tags). For nightly, we set NIGHTLY_BUILD=true and SHALLOW_CLONE=false (matching versions-nightly.env values).

4. **Update the packaging step** in each build job to also pass NIGHTLY_BUILD when the track is nightly. This is needed because `package_all.sh` checks NIGHTLY_BUILD:
   ```yaml
   - name: Package all components
     run: |
       export PATH="/opt/go/$(ls /opt/go/)/bin:$HOME/go/bin:$PATH"
       export DESTDIR="${{ runner.temp }}/podman-staging"
       if [[ "${{ steps.track.outputs.track }}" == "nightly" ]]; then
         export NIGHTLY_BUILD=true
       fi
       ./scripts/package_all.sh
   ```

5. **Update the Go cache key** for nightly builds. The current cache key uses `hashFiles('versions-stable.env')` which is irrelevant for nightly. Add a conditional:
   ```yaml
   - name: Cache Go modules and build cache
     uses: actions/cache@v4
     with:
       path: |
         ~/.cache/go-mod
         ~/.cache/go-build
       key: go-${{ runner.arch }}-${{ steps.track.outputs.track }}-${{ github.run_number }}
       restore-keys: |
         go-${{ runner.arch }}-${{ steps.track.outputs.track }}-
         go-${{ runner.arch }}-
   ```

6. **Update the publish job** to resolve the track and pass it to ci_publish.sh:
   ```yaml
   publish:
     needs: [build-amd64, build-arm64]
     runs-on: ubuntu-24.04
     environment:
       name: github-pages
       url: ${{ steps.deployment.outputs.page_url }}
     steps:
       - uses: actions/checkout@v4

       - name: Resolve build track
         id: track
         run: |
           TRACK="${{ inputs.build_track || 'nightly' }}"
           echo "track=${TRACK}" >> "$GITHUB_OUTPUT"

       - uses: actions/download-artifact@v4
         with:
           pattern: debs-*
           path: all-debs/
           merge-multiple: true

       - name: Install reprepro
         run: |
           sudo apt-get update
           sudo apt-get install -y reprepro

       - name: Build and publish repository
         env:
           GPG_PRIVATE_KEY: ${{ secrets.GPG_PRIVATE_KEY }}
         run: |
           ./scripts/ci_publish.sh \
             "${{ steps.track.outputs.track }}" \
             "all-debs" \
             "https://${{ github.repository_owner }}.github.io/${{ github.event.repository.name }}" \
             "repo-output"

       - uses: actions/configure-pages@v4
       - uses: actions/upload-pages-artifact@v3
         with:
           path: repo-output/
       - id: deployment
         uses: actions/deploy-pages@v4
   ```

7. **Remove the "Set version tags (stable)" step** from build jobs. The stable version parsing is already handled inside the "Build all components" step via the `while IFS= read` loop. The separate GITHUB_ENV step is redundant (and was the source of past bugs with quote stripping). Remove the `if: inputs.build_track == 'stable'` conditional step entirely from both build-amd64 and build-arm64.

IMPORTANT: Apply the same changes to BOTH build-amd64 and build-arm64 jobs (they are currently near-identical; keep them in sync).
