<objective>
Add nightly build track support to the build system and packaging scripts. This creates the foundational changes that allow building packages from the latest upstream commits (HEAD of main/master branches) rather than tagged releases, with correct Debian snapshot versioning.

Purpose: Enable bleeding-edge package builds that version correctly against tagged releases (nightly versions sort BELOW tagged releases via dpkg tilde convention), so users on the nightly track auto-upgrade when a real release lands.

Output: Modified build scripts, new nightly env file, nightly reprepro suite, and nightly-aware packaging.
</objective>

  <name>Task 2: Add nightly version extraction to package_all.sh</name>
  <files>scripts/package_all.sh</files>
  <behavior>
    - extract_version_nightly("podman", repo_path) returns a string matching pattern X.Y.Z~gitYYYYMMDD.XXXXXXX (7-char hex sha)
    - extract_version_nightly("pasta", repo_path) returns a plain YYYYMMDD datestamp (no tilde suffix)
    - extract_version_nightly("container-configs", repo_path) returns X.Y.Z~gitYYYYMMDD.XXXXXXX using common/ tag prefix
    - dpkg --compare-versions "${nightly_ver}~podman1" lt "${release_ver}~podman1" returns true (tilde sorts lower)
    - When NIGHTLY_BUILD=true, the packaging loop uses extract_version_nightly instead of extract_version for tag-based components
  </behavior>
  <action>
1. Add an `extract_version_nightly()` function to `scripts/package_all.sh` AFTER the existing `extract_version()` function. This function takes two args: `component` (string) and `repo_path` (path to the cloned repo). It extracts the development version from source files and appends `~git{YYYYMMDD}.{short_sha}`.

   Implementation per component (use the research's extraction patterns exactly):
   - **podman**: `grep 'RawVersion' "$repo_path/version/rawversion/version.go"` -> strip `-dev`
   - **buildah**: `grep '^[[:space:]]*Version[[:space:]]*=' "$repo_path/define/types.go"` -> strip `-dev`
   - **skopeo**: `grep 'Version[[:space:]]*=' "$repo_path/version/version.go"` -> strip `-dev`
   - **netavark, aardvark-dns**: `grep '^version[[:space:]]*=' "$repo_path/Cargo.toml"` -> strip `-dev`
   - **conmon**: `cat "$repo_path/VERSION"` -> trim whitespace
   - **fuse-overlayfs**: `grep 'AC_INIT' "$repo_path/configure.ac"` -> extract version, strip `-dev`
   - **catatonit**: `grep 'AC_INIT' "$repo_path/configure.ac"` -> extract version, strip `+dev`
   - **crun**: `git -C "$repo_path" describe --tags --abbrev=0 2>/dev/null || echo "0.0.0"`
   - **toolbox**: `grep "version:" "$repo_path/meson.build"` -> normalize 2-part to 3-part (append `.0`)
   - **container-configs**: `git -C "$repo_path" tag --list 'common/*' --sort=-version:refname | head -1 | sed 's|common/v||'`
   - **pasta**: Special case -- return plain `$(date +%Y%m%d)` with NO tilde suffix (already date-based)

   For all non-pasta components, output format: `${base_version}~git${datestamp}.${short_sha}`
   Where `short_sha=$(git -C "$repo_path" rev-parse --short=7 HEAD 2>/dev/null || echo "unknown")` and `datestamp=$(date +%Y%m%d)`.

   Fallback: if `base_version` is empty after extraction, set it to `"0.0.0"`.

2. Modify the packaging loop in `package_all.sh` to use `extract_version_nightly` when `NIGHTLY_BUILD=true`. In the loop where `local_tag` is resolved:

   After the pasta special case and the edge auto-detect block, add a nightly block:
   ```bash
   # Nightly builds: extract version from source files instead of tags
   if [[ "${NIGHTLY_BUILD:-false}" == "true" ]]; then
       local build_dir="${component}"
       if [[ -v "COMPONENT_BUILD_DIRS[$component]" ]]; then
           build_dir="${COMPONENT_BUILD_DIRS[$component]}"
       fi
       local_version="$(extract_version_nightly "${component}" "${BUILD_ROOT}/${build_dir}")${VERSION_SUFFIX}"
   else
       # ... existing tag-based version extraction ...
   fi
   ```

   Move the existing `local_version="$(extract_version ...)"` line into the `else` branch. The nightly branch skips tag validation entirely (there may be no tag at HEAD).

3. Similarly update the meta-package (podman-suite) section at the bottom. When `NIGHTLY_BUILD=true`, derive `suite_version` from `extract_version_nightly "podman" "${BUILD_ROOT}/podman"` instead of from the tag map.

  IMPORTANT: Do NOT change the existing `resolve_tag_from_repo()` function or `extract_version()` function. They continue to work for stable and edge tracks. The nightly path is a parallel code path.
